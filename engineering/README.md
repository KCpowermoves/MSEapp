# Engineering Calculator

Working folder for porting the MSE engineering Excel calculator into
the app. Everything in this folder except this README is `.gitignore`d
by default — the source workbook is reference data that may carry
real customer numbers, and inspector dumps are intermediate artifacts.

## Drop the workbook here

Save the Excel file as `source-calculator.xlsx` in this folder. The
inspector script (added in the next step) reads from that exact path.

```
engineering/
├── README.md                 # this file (committed)
├── source-calculator.xlsx    # YOUR FILE — gitignored
├── calculator.dump.md        # generated by inspector — gitignored
└── calculator.dump.json      # generated by inspector — gitignored
```

If you want to scrub customer numbers from the file before sharing,
keep a separate `source-calculator-anonymized.xlsx` and point the
inspector at it. Either works the same.

## Inspector workflow

1. Save the workbook to `engineering/source-calculator.xlsx`.
2. From the repo root, run `node engineering/inspect-calculator.mjs`
   (script added in the next implementation step).
3. The inspector walks every sheet, every cell, and emits:
   - `engineering/calculator.dump.md` — human-readable sheet-by-sheet
     table with formula + value side by side
   - `engineering/calculator.dump.json` — machine-readable for diffing
     against future revisions of the source workbook
4. From those dumps, the calculator logic gets ported to a TypeScript
   module under `lib/engineering/` and unit-tested by comparing
   outputs to the dumped values.

## Why we port instead of running the .xlsx server-side

We could run Excel formulas in Node via a library like
`hyperformula` — but the dependency footprint is huge and the
formula coverage is incomplete (some Excel functions just aren't
implemented). Porting once and keeping the .xlsx as a regression
baseline gives us cleaner code and tests we can trust.
